# Dead Time Machine 🌹⚡

Mobile-first Grateful Dead setlist browser. Tap an NFC tag → instant show lookup.
Setlists sourced and verified from **setlists.net**.

---

## What's in here

| File | What it does |
|---|---|
| `index.html` | The full web app — single file, no build step |
| `scrape_setlists.py` | Pulls all ~2,300 shows from setlists.net into the app |
| `shows.json` | Generated by the scraper (not included — run scraper) |

---

## Step 1 — Deploy to GitHub Pages (5 min)

```bash
# From this directory:
git init
git add index.html scrape_setlists.py README.md
git commit -m "dead time machine ⚡"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/dead-timemachine.git
git push -u origin main
```

Then in GitHub:
**Settings → Pages → Source: Deploy from branch → main → / (root) → Save**

Your URL: `https://YOUR-USERNAME.github.io/dead-timemachine/`

---

## Step 2 — Pull the full setlist archive (optional but recommended)

The app ships with 8 hand-verified shows. To get all ~2,300:

```bash
pip install requests beautifulsoup4
python scrape_setlists.py
```

This takes ~45 minutes (rate-limited to ~1 req/sec out of respect for setlists.net).
When done it auto-updates `index.html` with the full database and writes `shows.json`.

Then push the updated `index.html`:
```bash
git add index.html shows.json
git commit -m "full archive loaded"
git push
```

---

## Step 3 — Program the NFC tag

**App: NFC Tools** (free, iOS and Android)

1. Open NFC Tools → Write → Add a record → URL
2. Paste: `https://YOUR-USERNAME.github.io/dead-timemachine/`
3. Write → hold phone to tag

Stick the tag on your Time Machine. Done.

## Show of the Day

The home screen picks a show seeded by the current calendar date — changes automatically every day, same show for everyone on the same day. Once you have the full archive loaded, you'll get a different show every day for ~6 years before it cycles.

---

## Adding / editing shows

All data lives in the `ERAS` array near the top of the `<script>` block in `index.html`. Each show follows this shape:

```js
{
  id: "1009",               // setlists.net show_id
  date: "1977-05-08",       // YYYY-MM-DD
  dow: "Sunday",
  venue: "Barton Hall, Cornell University",
  city: "Ithaca, NY",
  badge: "Optional red callout line",
  sets: [
    { lbl: "Set One",  songs: ["Song A", "Song B > Song C"] },
    { lbl: "Set Two",  songs: ["Song D", "Song E"] },
    { lbl: "Encore",   songs: ["Last Song"], enc: true }
  ]
}
```

Segue notation: write as one string — `"Song A > Song B > Song C"`
